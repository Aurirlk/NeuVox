# 小智语音交互服务 - 部署手册

## 一、环境要求

### 1.1 服务器配置

- **操作系统**: Ubuntu 20.04+ / CentOS 7+ / Windows Server 2016+
- **CPU**: 2核+
- **内存**: 4GB+
- **磁盘**: 20GB+
- **网络**: 需要访问外部 API (DeepSeek, MiMo)

### 1.2 软件依赖

- Python 3.10+
- PostgreSQL 12+ (推荐) 或 SQLite (开发)
- Redis (可选，用于会话缓存)
- Nginx (推荐，用于反向代理)
- Docker + Docker Compose (容器化部署)

---

## 二、本地开发部署

### 2.1 环境准备

```bash
# 1. 克隆项目
git clone <repository_url>
cd XiaoZhi

# 2. 创建 conda 环境
conda create -n xiaozhi python=3.10
conda activate xiaozhi

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入 API Keys
```

### 2.2 启动服务

```bash
# 启动后端服务
python main.py

# 启动 Gradio UI (可选)
python voice_chat_ui.py
```

### 2.3 访问地址

- API 文档: http://localhost:8000/docs
- Gradio UI: http://localhost:7860
- WebSocket: ws://localhost:8000/ws/v1/chat/stream

## 三、Docker 部署

### 3.1 单容器部署

```bash
# 构建镜像
docker build -t xiaozhi-api .

# 运行容器
docker run -d \
  --name xiaozhi-api \
  -p 8000:8000 \
  -v $(pwd)/uploads:/app/uploads \
  -v $(pwd)/outputs:/app/outputs \
  -e DEEPSEEK_API_KEY=your_key \
  -e MIMO_TTS_API_KEY=your_key \
  xiaozhi-api
```

### 3.2 Docker Compose 部署

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 2. 启动所有服务
docker-compose up -d

# 3. 查看日志
docker-compose logs -f

# 4. 停止服务
docker-compose down
```

### 3.3 服务组件

| 服务          | 端口   | 说明             |
| ----------- | ---- | -------------- |
| xiaozhi-api | 8000 | FastAPI 后端     |
| postgres    | 5432 | PostgreSQL 数据库 |
| redis       | 6379 | Redis 缓存       |

## 四、生产环境部署

```bash

python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"
```

## 七、常见问题

### 7.1 API Key 错误

**问题**: `DEEPSEEK_API_KEY 未配置`

**解决**: 检查 `.env` 文件中的 API Key 是否正确

### 7.2 数据库连接失败

**问题**: `sqlalchemy.exc.OperationalError`

**解决**: 

- 检查 PostgreSQL 是否启动
- 检查 `DATABASE_URL` 配置
- 确保数据库用户有权限

### 7.3 端口被占用

**问题**: `Address already in use`

**解决**:

```bash
# 查找占用端口的进程
netstat -tlnp | grep 8000

# 终止进程
kill -9 <pid>
```

### 7.4 Docker 网络问题

**问题**: 容器间无法通信

**解决**:

```bash
# 重建网络
docker-compose down
docker-compose up -d
```

## 九、Postman 调试示例

### 9.1 导入 Collection

项目根目录已提供 `xiaozhi.postman_collection.json` 文件，导入步骤：

1. 打开 Postman
2. 点击左上角 **Import** 按钮
3. 选择 **File** 标签页
4. 选择 `xiaozhi.postman_collection.json` 文件
5. 点击 **Import** 完成导入

### 9.2 环境变量配置

导入后，在 Postman 右上角选择环境变量，设置：

| 变量名 | 值 | 说明 |
|--------|------|------|
| `base_url` | `http://127.0.0.1:8000` | 服务地址 |
| `session_id` | `test_session_001` | 会话ID |

### 9.3 调试场景

### 场景一：纯文本问答测试

**请求配置：**

| 字段           | 值                                  |
| ------------ | ---------------------------------- |
| Method       | POST                               |
| URL          | `http://127.0.0.1:8000/api/v1/ask` |
| Content-Type | multipart/form-data                |

**Body (form-data)：**

| Key        | Type | Value              |
| ---------- | ---- | ------------------ |
| session_id | Text | `test_session_001` |
| text       | Text | `你能用台湾腔做个自我介绍吗？`   |
| need_tts   | Text | `true`             |

**curl 命令：**

```bash
curl -X POST http://127.0.0.1:8000/api/v1/ask \
  -F "session_id=test_session_001" \
  -F "text=你能用台湾腔做个自我介绍吗？" \
  -F "need_tts=true"
```

**成功响应：**

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "session_id": "test_session_001",
        "recognized_text": null,
        "reply_text": "大家好～我是你的智能语音助手啦！很高兴认识你捏～",
        "audio_url": "/api/v1/assets/audio/resp_8a7b6.wav",
        "processing_time_ms": 1250
    }
}
```

### Postman Collection 文件

项目根目录提供 `xiaozhi.postman_collection.json` 文件，包含以下接口：

| 接口分组 | 接口名称 | 方法 | 说明 |
|----------|----------|------|------|
| 问答接口 | 文本问答 | POST | 发送文本消息进行问答 |
| 问答接口 | 语音问答 | POST | 上传语音文件进行问答 |
| 问答接口 | 文本问答 (不返回语音) | POST | 只返回文本，不生成语音 |
| 流式问答 | SSE 流式问答 | POST | LLM 流式输出 |
| 会话管理 | 清空会话历史 | DELETE | 清空指定会话 |
| 配置接口 | 健康检查 | GET | 检查服务状态 |
| 配置接口 | 获取服务提供商 | GET | 获取可用提供商列表 |
| 配置接口 | 获取音色列表 | GET | 获取 TTS 音色 |
| CRM 接口 | 获取 CRM 统计 | GET | 统计信息 |
| CRM 接口 | 获取用户列表 | GET | 用户列表 |
| CRM 接口 | 获取交互记录 | GET | 交互记录 |

**导入方法：**

1. 打开 Postman
2. 点击 **Import** → **File**
3. 选择 `xiaozhi.postman_collection.json`
4. 设置环境变量 `base_url` = `http://127.0.0.1:8000`
