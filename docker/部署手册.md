# NeuVox Docker 部署手册

## 一、环境要求

| 软件 | 版本 | 说明 |
|------|------|------|
| Docker | 20.10+ | 容器运行时 |
| Docker Compose | 2.0+ | 容器编排 |

## 二、快速部署

### 2.1 一键启动

```bash
# 进入 docker 目录
cd docker

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 2.2 服务列表

| 服务 | 端口 | 说明 |
|------|------|------|
| neuvox-api | 8000 | FastAPI 后端服务 |
| postgres | 5432 | PostgreSQL 数据库 |
| redis | 6379 | Redis 缓存 |
| chromadb | 8000 | ChromaDB 向量数据库 |

## 三、配置说明

### 3.1 环境变量

在项目根目录创建 `.env` 文件：

```bash
# API Keys
DEEPSEEK_API_KEY=your_key
MIMO_TTS_API_KEY=your_key

# 数据库
POSTGRES_PASSWORD=neuvox123

# 服务端口
PORT=8000
```

### 3.2 数据目录

```bash
# 项目根目录
├── uploads/          # 上传文件
├── outputs/          # 生成的音频
├── data/             # 数据库和知识库
│   ├── Knowledge/    # 知识库文件
│   └── neuvox.db     # SQLite 数据库
```

## 四、常用命令

```bash
# 启动服务
docker-compose up -d

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f neuvox-api

# 查看状态
docker-compose ps

# 进入容器
docker exec -it neuvox-api bash
```

## 五、访问地址

| 服务 | 地址 |
|------|------|
| API 文档 | http://localhost:8000/docs |
| 健康检查 | http://localhost:8000/api/v1/health |
| Gradio UI | http://localhost:7860 (需单独启动) |

## 六、数据持久化

### 6.1 数据卷

```yaml
volumes:
  postgres_data:    # PostgreSQL 数据
  redis_data:       # Redis 数据
  chroma_data:      # ChromaDB 向量数据
```

### 6.2 本地目录挂载

```yaml
volumes:
  - ../uploads:/app/uploads      # 上传文件
  - ../outputs:/app/outputs      # 生成音频
  - ../data:/app/data            # 数据库和知识库
```

## 七、故障排查

### 7.1 服务无法启动

```bash
# 检查日志
docker-compose logs neuvox-api

# 检查端口占用
netstat -tlnp | grep 8000
```

### 7.2 数据库连接失败

```bash
# 检查 PostgreSQL 状态
docker-compose ps postgres

# 重启数据库
docker-compose restart postgres
```

### 7.3 磁盘空间不足

```bash
# 清理 Docker 资源
docker system prune -a

# 查看数据卷
docker volume ls
```

## 八、生产环境建议

1. **HTTPS**: 配置 SSL 证书
2. **备份**: 定期备份 PostgreSQL 数据
3. **监控**: 添加健康检查和告警
4. **日志**: 配置日志收集（ELK/Loki）
5. **限流**: 配置 API 限流策略
